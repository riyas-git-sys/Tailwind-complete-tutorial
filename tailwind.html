Introduction to Tailwind CSS

Tailwind CSS is a utility-first CSS framework that allows you to build custom designs without ever leaving your HTML.

Key Benefits:

  • No more custom CSS to maintain
  • Responsive design made easy
  • Highly customizable
  • Small bundle size with PurgeCSS
  • Consistent design system

Setup & Installation

CDN Method (Quick Start)

<script src="https://cdn.tailwindcss.com"></script>

npm Installation (Recommended)

npm install -D tailwindcss
npx tailwindcss init

Then add Tailwind directives to your CSS:

@tailwind base;
@tailwind components;
@tailwind utilities;

Core Concepts

Utility-First Philosophy

Instead of predefined components, Tailwind provides low-level utility classes to build custom designs.

Traditional CSS Approach:

<!-- HTML -->

<button class="btn btn-primary">Click me</button>

<!-- CSS -->

.btn {

padding: 0.5rem 1rem;

border-radius: 0.25rem;

font-weight: bold;

}

.btn-primary {

background-color: #3b82f6;

color: white;

}

Tailwind CSS Approach:

<button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">

Click me

</button>

Responsive Design

Tailwind uses a mobile-first breakpoint system:

<!-- Width 16 by default, 32 on medium screens, 48 on large screens -->

<img class="w-16 md:w-32 lg:w-48" src="...">

Responsive Design

Tailwind uses a mobile-first breakpoint system. By default, styles apply to all screen sizes. Add responsive prefixes for larger screens:

<!-- Default: Extra small (xs) -->
<div class="w-16">...</div>

<!-- Small (sm) and up -->
<div class="w-16 sm:w-32">...</div>

<!-- Medium (md) and up -->
<div class="w-16 md:w-32">...</div>

<!-- Large (lg) and up -->
<div class="w-16 lg:w-32">...</div>

<!-- Extra large (xl) and up -->
<div class="w-16 xl:w-32">...</div>

Advanced Techniques

Custom Configuration

You can customize Tailwind by editing the tailwind.config.js file:

module.exports = {
content: ["./src/**/*.{html,js}"],
theme: {
extend: {
colors: {
'brand-blue': '#1fb6ff',
'brand-purple': '#7e5bef',
},
fontFamily: {
sans: ['Inter', 'sans-serif'],
},
},
},
plugins: [],
}

Using @apply to Create Components

Extract repeated utility patterns into CSS classes:

<style>

/* ... */

.btn {

@apply font-bold py-2 px-4 rounded;

}


/* ... */

.btn-blue {

@apply bg-blue-500 text-white;

}


/* ... */

.btn-blue:hover {

@apply bg-blue-700;

}

</style>

<button class="btn btn-blue"> Click me </button>

Dark Mode

Tailwind includes a dark mode variant:

<div class="bg-white dark:bg-gray-800"> <h1 class="text-gray-900 dark:text-white">Dark mode example</h1> <p class="text-gray-600 dark:text-gray-300">This text will change in dark mode</p> </div>
